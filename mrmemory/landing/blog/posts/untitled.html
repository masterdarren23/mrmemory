The Statelessness Problem

You're stuck rebuilding the same AI agent from scratch every time it runs. It's like trying to learn a new language without any prior knowledge – frustrating and inefficient. Stateless agents repeat mistakes, forget context, and waste computational resources. It's a common problem in AI development, but there's an easy fix.

Enabling Memory in CrewAI

To turn on memory in CrewAI, set memory=True when creating a Crew object:


from crewai import Crew
crew = Crew(
agents=[...],
tasks=[...],
memory=True # <--- That's it
)

This enables the memory system with sensible defaults — if you need more control, configure each memory type individually.

Understanding Memory Types

CrewAI's memory system has three types:

  1. Short-term memory: stores session context and is lost when the session ends.
  2. Long-term memory: stores cross-session learning and persists between runs.
  3. Entity memory: tracks people and concepts across sessions.

Configure each type using the Memory class:


from crewai import Memory
memory = Memory(
recency_weight=0.5,
recency_half_life_days=7
)

Using MrMemory for Persistent Knowledge Sharing

While CrewAI's memory system is powerful, implementing and managing it can be a challenge. That's where MrMemory comes in – a managed memory API that simplifies adding persistent knowledge sharing to your CrewAI agents:
