## Open-Source Weather Bot Cloned
- github.com/suislanchez/polymarket-kalshi-weather-bot → workspace/polymarket-kalshi-weather-bot/
- Python + FastAPI + React dashboard, MIT licensed
- **Core logic**: 31-member GFS ensemble from Open-Meteo (FREE) → count members above/below threshold → model probability → compare to market → trade when edge >8%
- Kelly sizing (15% fractional), max 5% bankroll per trade
- VALIDATED_RESEARCH.md corrects common mistakes:
  - NWS API = deterministic only (no probability ranges)
  - HRRR = deterministic only (need NBM or GEFS for probabilistic)
  - ECMWF open data = 50 members + 1 control (not 51)
  - NBM (National Blend of Models) = best US probabilistic source (~200 members)
- Recommended pipeline: NBM (percentiles) → ECMWF (50 members) → HRRR (short-range deterministic)
- Alternative: Open-Meteo Ensemble API (free REST, multiple models)
- 5 default cities: NYC, Chicago, Miami, LA, Denver
- Polymarket now has 56+ weather markets, Kalshi has 8-10 cities
